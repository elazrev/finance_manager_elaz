# תוכנית פיתוח - אפליקציית ניהול חשבונות לפטור עוסק

## טכנולוגיות עיקריות

### Frontend
- **Next.js 14** - Framework ראשי
- **TypeScript** - Type safety
- **Tailwind CSS** - Styling
- **Shadcn/ui** - Component library
- **React Query** - State management
- **Zod** - Validation
- **React Hook Form** - Form handling
- **Recharts** - Charts
- **React PDF** - PDF generation
- **date-fns** - Date manipulation

### Backend
- **Supabase** - Backend as a Service
  - PostgreSQL Database
  - Authentication
  - Storage
  - Functions
- **Row Level Security** - Authorization

### DevOps
- **Vercel** - Hosting & Deployment
- **Cloudflare** - CDN & DNS
- **GitHub Actions** - CI/CD
- **Sentry** - Error monitoring
- **Resend** - Email sending

## מבנה מסד נתונים

### טבלאות עיקריות
1. **users** - משתמשים והגדרות
2. **clients** - לקוחות
3. **invoices** - חשבוניות
4. **quotes** - הצעות מחיר
5. **items** - פריטים/שירותים

### יחסים
- User → Clients (1:N)
- User → Invoices (1:N)
- User → Quotes (1:N)
- User → Items (1:N)
- Client → Invoices (1:N)
- Client → Quotes (1:N)

---

## הערות חשובות

1. **חוקיות**: כל החשבוניות חייבות לעמוד בדרישות מס הכנסה בישראל
2. **אבטחה**: נתונים מוצפנים, RLS מופעל על כל הטבלאות
3. **ביצועים**: אופטימיזציה ל-queries, caching, lazy loading
4. **נגישות**: תמיכה בעברית מלאה, RTL, תאריכים עבריים
5. **גיבויים**: גיבוי יומי אוטומטי

## נקודות מפתח לפיתוח

1. **MVP ראשון**: התמקד בתכונות הליבה (לקוחות, חשבוניות, PDF)
2. **איטרציות**: בנה, בדוק, שפר - בכל שלב
3. **משוב משתמשים**: קבל משוב מוקדם ומשפר בהתאם
4. **תיעוד**: תיעד תוך כדי פיתוח
5. **בדיקות**: בדוק כל תכונה לפני מעבר לשלב הבא

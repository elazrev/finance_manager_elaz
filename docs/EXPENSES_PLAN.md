# תוכנית בנייה – ניהול הוצאות לעוסק פטור

## רקע משפטי

עוסק פטור **יכול לשקלל הוצאות** בחישוב מס הכנסה:
- הפטור הוא לעניין **מע"מ בלבד** – חייב בדיווח ותשלום מס הכנסה
- הוצאות מוכרות (סעיף 17 לפקודת מס הכנסה) מקטינות את ההכנסה החייבת
- יש לתעד הוצאות במסמכים תומכים למשך 7 שנים
- עוסק פטור מנכה **מלוא הסכום** כולל מע"מ – בניגוד לעוסק מורשה שמנכה רק את הבסיס

---

## 1. מודל נתונים – טבלת `expenses`

| שדה | סוג | חובה | תיאור |
|-----|-----|------|-------|
| id | UUID | ✓ | מפתח ראשי |
| user_id | UUID | ✓ | FK → users |
| amount | DECIMAL(12,2) | ✓ | סכום ההוצאה |
| date | DATE | ✓ | תאריך ההוצאה (לגביית מס – תאריך ביצוע) |
| category | VARCHAR(50) | ✓ | קטגוריה (מהרשימה המוגדרת) |
| supplier | VARCHAR(255) | | שם הספק/קבלן |
| description | TEXT | | תיאור ההוצאה |
| receipt_reference | VARCHAR(100) | | מספר חשבונית/קבלה |
| attachment_url | TEXT | | קישור למסמך סרוק (Supabase Storage) |
| vat_included | BOOLEAN | | האם הסכום כולל מע"מ (ברירת מחדל: true) |
| notes | TEXT | | הערות |
| created_at | TIMESTAMPTZ | | נוצר |
| updated_at | TIMESTAMPTZ | | עודכן |

## 3. שלבי בנייה מומלצים

### שלב א' – תשתית (Migration + API)
- [ ] Migration SQL: טבלת `expenses` + RLS + indexes
- [ ] Bucket ב-Supabase Storage: `expenses-receipts`
- [ ] API: `app/api/expenses/route.ts` – GET (list + filter), POST
- [ ] API: `app/api/expenses/[id]/route.ts` – GET, PATCH, DELETE
- [ ] API: העלאת קובץ ל-Storage (optional בשלב ראשון)

### שלב ב' – ממשק משתמש
- [ ] דף רשימת הוצאות: `/expenses` – טבלה, סינון לפי תאריך/קטגוריה
- [ ] דף הוספת הוצאה: `/expenses/new`
- [ ] דף עריכת הוצאה: `/expenses/[id]/edit`
- [ ] ניווט: קישור בתפריט הצד

### שלב ג' – אינטגרציה בדשבורד ודוחות
- [ ] Dashboard: אריח "סה״כ הוצאות החודש" + עדכון תצוגת רווח
- [ ] דוח הוצאות: סיכום לפי חודש/שנה, לפי קטגוריה
- [ ] דוח סיכום: הכנסות − הוצאות = רווח נקי
- [ ] ייצוא Excel: דף הוצאות + עדכון דוח הסיכום

### שלב ד' – תכונות מתקדמות (אופציונלי)
- [ ] העלאת תמונת קבלה/חשבונית מצורפת
- [ ] תבניות הוצאות חוזרות
- [ ] חיבור הוצאה לחשבונית (אם רלוונטי)

## 4. לוגיקת חישוב – עקביות עם עוסק פטור

| חישוב | תאריך רלוונטי | הערה |
|-------|----------------|------|
| סינון הוצאות לפי חודש/שנה | `date` (תאריך ההוצאה) | תואם לדיווח מס – תאריך ביצוע |
| רווח לשנה | הכנסות (paid_at) − הוצאות (date) | שני התאריכים באותה שנה |
| סיכום חודשי | `date` בטווח | לדוחות פנימיים |

## 7. הערות טכניות

- **Storage**: bucket `expenses-receipts` עם RLS – גישה רק ל-`user_id` המתאים  
- **דחיסת תמונות**: ראו `docs/RECEIPT_STORAGE_STRATEGY.md` – דחיסה עם Sharp לפני העלאה מקטינה גודל ל־~80–200 KB לצילום  
- **Categories**: ניתן לאחסן כ-enum או JSON ב-settings – בשלב ראשון enum בקבצים  
- **i18n**: כל הטקסטים בעברית – אין צורך ב-i18n נוסף כרגע
